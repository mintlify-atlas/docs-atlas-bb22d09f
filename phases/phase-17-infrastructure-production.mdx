## Completed lessons

### Model serving (lesson 01 — complete)

Model serving is the problem of taking a trained model and making it available to callers via a network API with acceptable latency, throughput, and cost. Key topics:

- REST vs gRPC for inference endpoints
- Batching: static batching, dynamic batching, continuous batching
- Caching: KV cache for autoregressive models, prompt caching for repeated prefixes
- Serving frameworks: vLLM, TGI, Triton Inference Server
- Quantization at inference time: INT8, GPTQ, AWQ for lower memory and faster decoding

### Docker for AI workloads (lesson 02 — complete)

AI workloads have specific container requirements: GPU drivers, CUDA libraries, large model weights, and long startup times. Lesson 02 covers:

- Multi-stage Dockerfiles that separate build-time dependencies from runtime
- NVIDIA Container Toolkit for GPU access inside containers
- Volume mounts for model weight caching (avoid re-downloading on each start)
- Health checks for inference endpoints
- Rust is used for a sidecar process that monitors container resource usage

### Kubernetes for AI (lesson 03 — complete)

Running AI workloads on Kubernetes introduces GPU scheduling, pod disruption budgets, and custom resource definitions. Key topics:

- Node pools with GPU labels and taints
- `requests` and `limits` for GPU resources (`nvidia.com/gpu`)
- Horizontal pod autoscaling based on inference queue depth
- Persistent volumes for shared model weight storage
- KubeRay for distributed training and serving

## What breaks in production

Production AI systems fail in ways that development environments do not reveal:

| Failure mode | Symptom | Fix |
|-------------|---------|-----|
| Context window overflow | Errors on long conversations | Implement context compression (Phase 14 lesson 06) |
| GPU memory exhaustion | OOM errors under load | Tune batch size, use quantization |
| Flaky external tools | High tool error rate | Retry with backoff, circuit breakers |
| Prompt injection | Unexpected agent behavior | Input sanitization, output filtering |
| Cost overrun | Unexpected bill | Token budgets, model routing, caching |
| Latency regression | Slow responses after model update | Canary deployments, latency SLOs |
