## Core concept: convolution as a sliding dot product

Convolution is the mathematical heart of CNNs. A kernel slides across an input feature map, computing a dot product at each position. The result is a new feature map that highlights the pattern the kernel was trained to detect.

```python
import numpy as np

def convolve2d(image, kernel, stride=1, padding=0):
    """Manual 2D convolution — no frameworks."""
    if padding > 0:
        image = np.pad(image, padding, mode='constant')

    kH, kW = kernel.shape
    iH, iW = image.shape
    oH = (iH - kH) // stride + 1
    oW = (iW - kW) // stride + 1
    output = np.zeros((oH, oW))

    for i in range(0, oH):
        for j in range(0, oW):
            patch = image[i*stride:i*stride+kH, j*stride:j*stride+kW]
            output[i, j] = np.sum(patch * kernel)

    return output

# Sobel edge-detection kernel
sobel_x = np.array([[-1, 0, 1],
                     [-2, 0, 2],
                     [-1, 0, 1]], dtype=float)

image = np.random.rand(8, 8)
edges = convolve2d(image, sobel_x, padding=1)
print(f"Input shape:  {image.shape}")
print(f"Output shape: {edges.shape}")
```

In practice, PyTorch's `nn.Conv2d` implements the same operation for batches of multi-channel feature maps, with learnable kernels optimized by backpropagation.
