## Key concepts

### Function calling vs tool use

Function calling is how an LLM outputs structured JSON instead of free text — signalling that it wants to invoke a specific function with specific arguments. Tool use is the broader pattern: registering a set of callable functions with the model, parsing the LLM's structured response, executing the function, and feeding the result back.

Every agent framework builds on this loop. Phase 13 covers it from the API wire format all the way to patterns for retry, parallel tool calls, and schema validation.

### Model Context Protocol (MCP)

MCP is Anthropic's open standard for connecting LLMs to external data sources and tools. Instead of hardcoding tools into every application, MCP separates concerns: a **server** exposes capabilities (tools, resources, prompts) and a **client** (the agent) discovers and calls them at runtime.

This matters because:
- Tools are reusable across agents and applications
- Servers can be built once and shared (e.g., a filesystem server, a GitHub server, a database server)
- Clients do not need to know tool schemas at compile time

MCP defines three primitives:

| Primitive | What it is |
|-----------|------------|
| **Tools** | Functions the LLM can call (read file, run query, send email) |
| **Resources** | Data the LLM can read (file contents, database rows, API responses) |
| **Prompts** | Reusable prompt templates with dynamic arguments |

### API design for AI

Designing APIs for LLMs is different from designing APIs for humans. LLMs need:
- Precise, unambiguous parameter names and descriptions
- Narrow input/output schemas (avoid `any` or open-ended objects)
- Error messages that explain what went wrong and how to fix it
- Idempotent operations where possible (agents retry on failure)

Lesson 08 covers these patterns in depth, including how to document your API so a model can use it correctly without human guidance.

Lesson 09 covers Playwright-based browser control: navigating pages, clicking elements, filling forms, extracting structured data, and handling dynamic JavaScript-heavy sites. You will build a web agent capable of autonomous research across multiple pages.
