Phase 3 is the conceptual core of the entire course. The rule here is simple: no frameworks until you've built what they abstract. You'll start with the original perceptron, derive the multi-layer network forward pass by hand, implement backpropagation from the chain rule alone, then build every standard component — activations, loss functions, optimizers, regularization — in pure Python. The capstone is lesson 10: a working mini deep learning framework (~500 lines, no dependencies) that trains a neural network on a real classification problem. Only after building it yourself do lessons 11 and 12 introduce PyTorch and JAX, at which point the framework code will feel familiar rather than magical.

## Key concepts

  <Card title="The perceptron & forward pass" icon="arrow-right">

  </Card>
  <Card title="Backpropagation" icon="arrow-left">
    Reverse-mode automatic differentiation applied to a computation graph. The chain rule iterated layer by layer to compute `∂L/∂W` for every parameter simultaneously — the algorithm that made deep learning tractable.
  </Card>
  <Card title="Activation functions" icon="wave-square">
    ReLU and its dead-neuron problem. Sigmoid and vanishing gradients. GELU's smooth approximation used in modern transformers. How activation choice affects gradient flow depth.
  </Card>
  <Card title="Loss functions" icon="target">
    MSE for regression, binary and categorical cross-entropy for classification, contrastive loss for similarity learning. How each loss shapes the gradient signal the optimizer receives.
  </Card>

## Lesson 10 deep dive: Build Your Own Mini Framework

Lesson 10 is the Phase 3 capstone. You build a working deep learning framework in ~500 lines of pure Python — no NumPy, no PyTorch, no dependencies. The framework includes every component built in lessons 01–09, integrated into a coherent, PyTorch-style API.

### Framework architecture

```
phases/03-deep-learning-core/10-mini-framework/code/main.py
├── Module (base class: forward, backward, parameters, train/eval)
├── Layers
│   ├── Linear     — weight matrix + bias, He initialization
│   ├── ReLU       — element-wise, stores mask for backward
│   ├── Sigmoid    — numerically stable, stores output for backward
│   ├── Tanh       — stores output for backward
│   ├── Dropout    — inverted dropout with training/eval toggle
│   └── BatchNorm  — running stats, gamma/beta learnable params
├── Containers
│   └── Sequential — composes modules, routes forward/backward
├── Loss functions
│   ├── MSELoss    — mean squared error
│   └── BCELoss    — binary cross-entropy, numerically stable
├── Optimizers
│   ├── SGD        — with optional momentum
│   └── Adam       — bias-corrected first/second moment estimates
└── DataLoader     — batching + shuffle for mini-batch training
```

### Real code: the `Sequential` container and training loop

The `Sequential` module is the heart of the framework — it chains any number of `Module`s together and routes gradients in reverse order during backprop:

```python
# From phases/03-deep-learning-core/10-mini-framework/code/main.py

class Sequential(Module):
    def __init__(self, *modules):
        super().__init__()
        self.modules = list(modules)

    def forward(self, x):
        for module in self.modules:
            x = module.forward(x)
        return x

    def backward(self, grad):
        # Reverse order: gradients flow from output back to input
        for module in reversed(self.modules):
            grad = module.backward(grad)
        return grad

    def parameters(self):
        params = []
        for module in self.modules:
            params.extend(module.parameters())
        return params

    def train(self):
        self.training = True
        for module in self.modules:
            module.train()

    def eval(self):
        self.training = False
        for module in self.modules:
            module.eval()


# Training a 4-layer network on the circle classification task
model = Sequential(
    Linear(2, 16),
    ReLU(),
    Linear(16, 16),
    ReLU(),
    Linear(16, 8),
    ReLU(),
    Linear(8, 1),
    Sigmoid(),
)

criterion = BCELoss()
optimizer = Adam(model.parameters(), lr=0.01)

model.train()
for epoch in range(100):
    for x, t in train_data:
        pred = model.forward(x)
        loss = criterion(pred, t)

        optimizer.zero_grad()
        grad = criterion.backward()
        model.backward(grad)
        optimizer.step()
```

### What the framework proves

Running the three experiments in `main.py` shows real results from a purely Python implementation:

| Experiment | Architecture | Optimizer | Test accuracy |
|------------|-------------|-----------|:-------------:|
| Baseline | 2→16→16→8→1 | Adam (lr=0.01) | ~95% |
| SGD comparison | Same | SGD (lr=0.1) | ~88% |
| With Dropout | Same + Dropout(p=0.3) | Adam (lr=0.01) | ~94% |

These numbers come from 500 samples on a circle classification task — inside or outside a circle of radius √1.5 — trained for 100 epochs in pure Python with no acceleration.
