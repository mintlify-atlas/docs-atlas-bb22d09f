Phase 0 is the foundation that every subsequent lesson depends on. Before writing a single neural network layer, you need a reproducible, professional-grade development environment. These 12 lessons walk you through every layer of the modern AI developer's stack — from terminal and Linux basics through GPU provisioning, containerized workflows, and Python dependency management. By the end, your machine (or cloud instance) will be configured exactly as it is throughout all 260+ lessons of the course.

<Note>
All 12 Phase 0 lessons are complete (✅). You can work through them at any pace — each lesson stands alone and produces a working setup artifact.
</Note>

## Key concepts

<CardGroup cols={2}>
  <Card title="Reproducible environments" icon="box">
    Virtual environments (`venv`, `conda`) and Docker containers ensure your code runs identically on every machine, cloud instance, or CI runner.
  </Card>
  <Card title="GPU provisioning" icon="microchip">
    Setting up CUDA, selecting cloud GPU instances (AWS, GCP, Lambda Labs), and verifying driver compatibility before Phase 3's neural network training.
  </Card>
  <Card title="Secure API management" icon="key">
    Storing secrets in `.env` files, loading them with `python-dotenv`, and never committing keys to version control — a pattern used across every phase that calls an AI API.
  </Card>
  <Card title="Debugging & profiling" icon="bug">
    Using `pdb`, `ipdb`, `cProfile`, `line_profiler`, and `py-spy` to find performance bottlenecks — critical when training loops are slow or models produce unexpected outputs.
  </Card>
</CardGroup>

## What you'll ship from this phase

Every lesson produces a reusable artifact. From Phase 0 you'll have:

- A **`Dockerfile` and `docker-compose.yml`** for AI workloads, reusable across all phases
- A **`.env` key management template** compatible with every API lesson
- A **Python environment setup script** that installs the full course dependency stack
- A **profiling toolkit** (`profile.py`) ready to plug into any training loop

```bash
# Verify your Phase 0 setup is complete
python --version        # 3.10+
nvidia-smi              # GPU visible
docker run hello-world  # Docker working
jupyter --version       # Jupyter installed
```

## Getting started with Phase 0

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/rohitg00/ai-engineering-from-scratch.git
    cd ai-engineering-from-scratch
    ```
  </Step>
  <Step title="Open the first lesson">
    ```bash
    cd phases/00-setup-and-tooling/01-dev-environment
    cat docs/en.md
    ```
    Read the lesson doc, then follow the Build It steps.
  </Step>
  <Step title="Install the base requirements">
    ```bash
    python -m venv .venv
    source .venv/bin/activate   # Windows: .venv\Scripts\activate
    pip install -r requirements.txt
    ```
  </Step>
  <Step title="Work through lessons 02–12 in order">
    Each lesson builds on the previous one. Complete them in sequence to arrive at a fully configured environment by lesson 12.
  </Step>
  <Step title="Check your understanding">
    Use the built-in Claude Code skill to quiz yourself before moving to Phase 1:
    ```
    /check-understanding 0
    ```
  </Step>
</Steps>
